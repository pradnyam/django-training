Some history about Django

Django is python based web framework; created for fast paced and database driven web site. This most secure, stable and flexible python framework till date. And also have well written documentation, which can be found at Django Documentation Current version available is 1.10; Django is compatible with both Python 2.X and 3.X versions.

In fall of 2003, at World Online, the Web department of a newspaper in Lawrence, Kansas, USA, web developers were working on python applications to meet fast moving newsroom deadlines. This is when Django was born. Initially it was launched with BSD license, now it is being handled by Django Software Foundation (DSF) team (Non profit organization).

Why to use Django/ Features provided by Django

1. Superfast development
Django has all the components available, you don’t need to actually write every component. You just to overload the functionalities to apply business logic.
Since Django is an open source technology and it has huge developers base; you can find multiple Django application available suiting your requirements.

For example if you want to have multilingual application you can use Django-Rosetta which handles your word to word translations. Django-Rosetta is one of the available application for internationalization. There are a lot of other alternative to this application.

You can find multiple available packages for any category at https://djangopackages.org

2. Batteries included
Django framework’s main philosophies are reusability and pluggability of the components. Django has lots of inbuilt components and also have 3rd party applications available.

Inbuilt components includes ORM, templating Engine, Authentication schemes, Admin Console, translation support, site map, RSS Feeds and many more.

3. Very Scalable
Scalability can be measured in terms of robustness and extensibility. The framework clearly separates components such as its database layer and application layer. If you make any changes in views it would have minimum impact at templates level.

Django supports multiple databases; if we have primary-secondary databases. You can create simple Python class called as Database Router, using which you can decide which database can be used for which functionality. And it ships with a simple-yet-powerful cache framework. https://blog.disqus.com/scaling-django-to-8-billion-page-views

4. Secure and Stable
Django handles multiple security concerns. Like Cross site request forgery protection, SQL injection, Clickjacking protection and also supports SSL/HTTPS.

Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated. CSRF attacks specifically target state-changing requests, not theft of data, since the attacker has no way to see the response to the forged request.

5. Flexible
Every component in Django Framework is customizable hence making it very flexible. You can customize any module of the Framework.

6. Incredibly versatile
Django is used across the domains, here are some of the popular sites which uses Django

Django Fundamentals

1. MVT (Model-View-Template)
It is design pattern similar to MVC, just names are shuffled.

2. DRY (Don’t Repeat Yourself)
Use inheritance and extend the models/views/templates/forms to have the common functionalities.

3. Loose Coupling
Django ORM do not execute the query when it called but when it is being used. E.g. If users= User.objects.all() at the query won’t be called but it would be called when we iterate over list of the users.

4. Extensibility
Use inheritance or user 3rd party application before writing your custom code.

5. Simplicity
Django/python is very high level language with English meaning statements.

6. Less Code
There a lot of inbuilt components available hence try to reuse the code, and code is expected to be written is business logic.
